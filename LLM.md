# @zap-proto/zap — AI Assistant Context

This repo (`zap-proto/ts`) is the **ZAP foundation layer** for TypeScript,
published as `@zap-proto/zap`. It is the Cap'n-Proto wire codec + Level-4 RPC
primitives + the `capnpc-*` compiler. Fork of `unjs/capnp-es`.

## Decomplect — what lives where

- `@zap-proto/zap` (this repo, `zap-proto/ts`) — foundation only: codec, RPC,
  compiler. NO MCP, NO browser glue, NO app protocols.
- `@zap-proto/mcp` (`zap-proto/mcp`) — Model Context Protocol over ZAP. Depends
  on `@zap-proto/zap`. (Was previously mis-published here as `@zap-proto/zap`
  v1.0.1; migrated out to its own repo.)
- `@zap-proto/web` — built separately on top of `@zap-proto/zap/rpc`. Not here.

## Source layout

- `src/codec/serialization/` — wire runtime (message, segment, arena, pointers, packing)
- `src/codec/capnp/` — generated built-in schema bindings (regenerated by `build:std`)
- `src/rpc/` — Level-4 RPC primitives (Conn, Client, Server, Pipeline, …)
- `src/compiler/` — `capnpc-ts` / `capnpc-js` / `capnpc-dts` generators
- `src/debug/` — message dump helpers
- `src/{constants,errors,util}.ts` — shared primitives across codec + rpc + compiler

## Entry points (package.json `exports`)

- `.` — full runtime (codec + rpc). Generated code does `import * as $ from "@zap-proto/zap"`.
- `./rpc` — RPC only (focused, for consumers like `@zap-proto/web`)
- `./debug`, `./compiler`, `./codec/capnp/*`

## Conventions

- Generated `codec/capnp/*.ts` import the runtime by package name `@zap-proto/zap`.
  The default `capnp` runtime import string lives in
  `src/compiler/generators/imports.ts`.
- `pnpm build` = `build:std` (stub + regenerate `codec/capnp` via compiler) then `unbuild`.
- Tag `vX.Y.Z` to publish — CI publishes, never `npm publish` by hand.
